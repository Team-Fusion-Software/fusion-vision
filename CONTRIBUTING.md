# Contributing

Thanks for contributing to this Team Fusion Software repository. To keep `main` stable and reviewable, **every** change goes through a Pull Request that a maintainer reviews and approves before it is merged.

## Workflow

1. **Clone** the repository:
   ```
   git clone https://github.com/Team-Fusion-Software/fusion-vision.git
   ```
2. **Create a feature branch** off `main`:
   ```
   git checkout -b my-feature
   ```
3. **Make your changes** and commit them with clear, descriptive messages.
4. **Push** your branch to GitHub:
   ```
   git push origin my-feature
   ```
5. **Open a Pull Request** into `main`.
6. **Wait for maintainer review and approval.** At least one approving review from a Code Owner is required before merging.
7. Once approved, the Pull Request can be merged into `main`.

## Rules

- No direct pushes to `main`. The `main` branch is protected; all changes must arrive via a Pull Request.
- No force pushes to `main`.
- Every Pull Request requires at least one approving review from a maintainer (Code Owner) before it can be merged.
- Keep Pull Requests focused and reasonably small so they are easy to review.
